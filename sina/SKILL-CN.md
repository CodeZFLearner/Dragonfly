# Skill: 查询公司财务报表 (Finance Data Query)

## 1. 工具概述
**名称**: `query_finance_data`  
**接口文档**: http://localhost:8080/v3/api-docs
**描述**: 该工具用于从财经数据源查询上市公司的财务报表数据。支持查询关键指标、资产负债表、利润表及现金流量表。用户可通过指定股票代码、报表类型及分页参数获取结构化的财务数据。  
**适用场景**: 金融数据分析、个股基本面研究、自动化财报监控、财务指标对比。

## 2. 输入参数规范 (Input Schema)

调用此工具时，需构建一个包含以下字段的查询对象 (`SinaFinanceQueryDTO`)。所有参数均通过 URL Query 形式传递。

### 核心参数

| 参数名 | 类型 | 必填 | 默认值 | 描述与约束                                                                             | 示例值 |
| :--- | :--- | :--- | :--- |:----------------------------------------------------------------------------------| :--- |
| **`code`** | `string` | **是** | - | **股票代码**。通常包含市场前缀（如 `sh` 代表上海，`sz` 代表深圳）。                                         | `sh600519`, `sz000001` |
| **`source`** | `string` | 否 | `gjzb` | **数据来源标识**。决定返回哪类报表数据。• `gjzb`: 关键指标 (默认)• `fzb`: 资产负债表• `lrb`: 利润表• `llb`: 现金流量表 | `lrb` (查询利润表) |
| **`type`** | `integer` | 否 | `0` | **报告期类型**。• `0`: 全部 (默认)• `1`: 一季报• `2`: 中报 (半年报)• `3`: 三季报• `4`: 年报              | `4` (仅查询年报) |
| **`page`** | `integer` | 否 | `1` | **页码**。最小值为 1，用于分页获取大量数据。                                                         | `2` |
| **`num`** | `integer` | 否 | `10` | **每页条数**。范围 1-100。建议设置为 20-50 以平衡性能与信息量。                                          | `20` |

## 3. 使用指南 (Usage Guide)

### 3.1 基础查询：获取最新关键指标
当用户仅想知道某只股票的基本财务概况时，只需提供股票代码，其他参数使用默认值（关键指标、全部报告期）。

*   **用户指令**: "帮我查一下贵州茅台的财务数据。"
*   **工具调用参数**:
    ```bash
    curl -X 'GET' \
    'http://localhost:8080/sina/finance/data?code=600519&source=gjzb&type=0&page=1&num=10' \
    -H 'accept: */*'
    ```

### 3.2 进阶查询：获取特定报表（如利润表）
当用户需要分析公司的盈利能力时，需指定 `source` 为 `lrb`。

*   **用户指令**: "我想看腾讯控股去年的利润表。"
*   **工具调用参数**:
    ```bash
    curl -X 'GET' \
    'http://localhost:8080/sina/finance/data?code=hk00700&source=lrb&type=4&page=1&num=20' \
    -H 'accept: */*'
    ```
    *(注：若代码格式不确定，可尝试去除前缀或根据常见格式推断，如腾讯港股通常为 `hk00700` 或类似格式，具体需依数据源实际支持情况)*



## 4. 最佳实践 (Best Practices)
*   **批量查询优化**: 如果用户要求对比多只股票（如“对比茅台和五粮液”），应并发调用工具获取各股票数据，然后进行横向对比分析。
*   **上下文记忆**: 记住用户当前关注的股票代码。如果用户接着问“那它的现金流怎么样？”，无需再次询问代码，直接使用上下文中保存的 `code`，并将 `source` 切换为 `llb` 进行查询。
